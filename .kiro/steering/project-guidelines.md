# Project Guidelines

## Documentation Rules

1. **Test First**: NEVER write documentation before testing the code works
2. **Minimal Documentation**: Only ONE README.md per project - no excessive documentation files
3. **No Redundant Files**: Don't create multiple guides (QUICK_START, RUN_LOCAL, SUCCESS, etc.)
4. **Keep It Simple**: README should be concise and practical

## Code Standards

- Test all code changes before committing
- Use meaningful variable names
- Follow existing code patterns
- Add comments only for complex logic

## Security

- Never commit credentials to git
- Use `-local.json` suffix for files with real credentials
- Add sensitive files to .gitignore
